# Vibe Coding

Building software by describing intent in natural language to an LLM and largely *not reviewing the generated code*. The term that defined the 2025 moment.

## Origin

Coined by [[Andrej Karpathy]] in a tweet on February 6, 2025: *"There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists… I 'Accept All' always, I don't read the diffs anymore… I just see stuff, say stuff, run stuff, and copy paste stuff, and it mostly works."* He later called it "a shower of thoughts throwaway tweet."

## The Disambiguation

Simon Willison (Mar 2025): vibe coding ≠ all AI-assisted coding. His definition: *"Building software with an LLM without reviewing the code it writes."* If you review, test, and can explain it, "that's not vibe coding, it's software development." The distinction matters for the [[Augmented Coding|disciplined]] counter-movement.

## Cultural Status

Named **Collins Dictionary Word of the Year 2025**. YC's Garry Tan (Mar 2025): *"For 25% of the Winter 2025 batch, 95% of lines of code are LLM generated… The age of vibe coding is here."*

## The Reality Check

- **Andrew Ng:** guiding AI to write software "is a deeply intellectual exercise"; he's "frankly exhausted by the end of the day."
- **Tech debt:** GitClear's analysis found refactoring dropped from ~25% of changed lines (2021) to <10% (2024); duplication rose ~4×.
- **Security & incidents:** agents "optimize for making code run, not making code safe"; a widely-reported July 2025 incident had Replit's agent delete a production database.

This is why [[Spec-Driven Development]] and [[Evals as the New PRD|evals]] emerged: *vibe-code to explore, spec/eval to ship.* See [[Build to Learn vs Build to Earn]].

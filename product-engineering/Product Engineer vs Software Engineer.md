# Product Engineer vs Software Engineer

[[PostHog]]'s one-liner (Vanagas & Yoon, updated Dec 2025): *"Product engineers focus on building great products while software engineers focus on building reliable software."*

## The Distinction

| | Product Engineer | Software Engineer |
|---|---|---|
| Owns | The product (its successes and failures) | The code (its correctness and reliability) |
| Optimizes for | User outcomes, iteration velocity | The ideal technical solution, scale, security |
| Disposition | Pragmatic, generalist, "build fast even from scratch" | Idealistic, often more specialized |
| Activities | Talks to customers, reads usage data, scans competitors | Architecture, reliability, performance |

A product engineer is *not* just a [[The Generalist|full-stack engineer]]: full-stack describes technical breadth (front + back), while product engineering adds *product* breadth — pricing, revenue, UX, and direct customer contact. "Product engineers go beyond shipping code; they engineer every aspect of product success."

## The AI Angle

[[PostHog]]: with LLMs, "product engineers can use LLMs to quickly move features from 0→1 without getting bogged down by implementation details." As AI makes reliable code cheaper to produce, the differentiator shifts from *writing the code* toward *knowing what's worth building* — see [[Taste as the Bottleneck]] and [[Will AI Replace Engineers]].

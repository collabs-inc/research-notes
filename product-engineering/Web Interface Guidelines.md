# Web Interface Guidelines

A checklist of interaction, animation, layout, form, and performance details that "make a good web interface," created by [[Rauno Freiberg]] (Staff Design Engineer at [[Vercel]]). Adopted by Vercel as an official resource (vercel.com/design/guidelines) and published openly on GitHub, installable via CLI.

## Why It Matters

The Guidelines are one of the clearest artifacts of the [[Craft as a Moat|craft]] movement: they make *taste explicit and reusable*. Rather than living in one designer's head, detail-level quality becomes a shared, versioned reference an entire org — or an AI agent — can apply.

This is part of a pattern in which the [[Design Engineer|design-engineer]] diaspora spreads craft through *artifacts* rather than only through hiring:

- [[Rauno Freiberg]] — Web Interface Guidelines, *Devouring Details* (interactive interaction-design manual, 2025)
- [[Emil Kowalski]] — *Animations on the Web* course; a "Design Engineer" skill packaging craft heuristics for AI coding agents
- [[Paco Coursey]] — cmdk; [[shadcn]] — shadcn/ui

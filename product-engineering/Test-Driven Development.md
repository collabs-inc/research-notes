# Test-Driven Development

The discipline of writing a failing test before the code that makes it pass, created by [[Kent Beck]] as part of Extreme Programming. Included here because of its renewed relevance in the agentic era.

## Why It Matters Now

[[Kent Beck]] argues TDD is a "superpower" for [[Augmented Coding|augmented coding]]: when an AI "genie" writes the code, a tight suite of tests is what keeps it honest — catching regressions and preventing it from, say, deleting tests to make them pass. The old practice becomes *more* valuable as agents write more code, not less.

TDD is also the conceptual ancestor of [[Evals as the New PRD|evals]]: both encode "what good looks like" as an executable check that runs continuously. For deterministic code, that's a unit test; for non-deterministic AI behavior, it's an eval.
